In dieser Aufgabe musst Du die Wahrscheinlichkeit ein Brettspiel zu gewinnen berechnen. Das Spiel besteht aus zwei Spielern die Einheiten über eine Karte bewegen. Wenn sich Einheiten bekämpfen, würfeln beide Spieler um den Sieg, ein Wurf für jede Einheit. Wir wollen die Wahrscheinlichkeit eines Sieges in einer solchen Schlacht wissen, ungeachtet eigener Verluste. Denke daran, falls die Schlacht ended ohne verbleibende Einheiten auf beiden Seiten, ist das ein Unendschieden, kein Sieg.

Sowie eine Schlacht beginnt, würfelt jeder Spieler einmal pro Einheit. Jeder Würfel hat eine Anzahl von Angriffs- und Verteidigungszeichen auf jeder Seite. Nach jeder Runde, zieht man die Anzahl gewürfelter Verteidigunszeichen von den gegnerischen Angriffszeichen ab, und hat bei einem positven Ergebnis entsprechend viele Einheiten verloren. Würfelt ein Spieler zum Beispiel 2 Angriffe und 4 Verteidigungen, sein Gegner 3 Angriffe und 1 Verteidigung, hat er keine Verluste (3 - 4) und sein Gegner würde 1 Einheit verlieren (2 - 1).

Nachdem die verlorenen Einheiten entfernt sind, würfeln die Spieler erneut. Solange bis ein Spieler keine Einheiten mehr hat.

Du erhälst eine Beschreibung des Würfels, als eine Liste mit den Aktionszeichen auf jeder Seite, und wieviele Einheiten jeder Spieler hat. Die Würfel beider Spieler sind genau gleich. Jedes Element der Liste ist ein String mit keinem oder mehreren A's als Angiffszeichen (Attack), und keinem oder mehreren D's, welche die Verteidigung (Defense) darstellen. (Eine Seite kann ganz leer sein.) Zum Beispiel die Liste ["AAD", "ADD", "A", "D", "", ""] zeigt einen sechsseitigen Würfel mit zwei Angriffen und einer Verteidigung auf einer Seite, einem Angriff mit zwei Verteidigungen auf der nächsten, einem einzelnen Angriff, einer Verteidigung und zwei leeren Seiten.

Du musst die Wahrscheinlichkeit für einen Sieg von Spieler Eins berechnen. Wenn Spieler Eins eine 1 zu 7 Chance zu gewinnen hat, gibst Du den Wert ≈0.1429 zurück. Das Ergebnis sollte also auf vier Nachkommastellen, ±0.0001, gerundet sein.

Eingabe: Drei Argumente. Eine Beschreibung des Würfels als eine Liste von Strings. Die Anzahl der Einheiten von Spieler Eins und Zwei als Integer.

Ausgabe: Die Wahrscheinlichkeit, dass Spieler Eins die Schlacht gewinnt als Float oder Integer.

Beispiele: 

battle_probability(['A', 'D'], 3, 3) == 0.0000 # Jeder Spieler verliert immer dieselbe Anzahl Einheiten.
battle_probability(['A', 'D'], 4, 3) == 1.0000
battle_probability(['AA', 'A', 'D', 'DD'], 3, 4) == 0.0186
battle_probability(['AA', 'A', 'D', 'DD'], 4, 4) == 0.4079
battle_probability(['AA', 'A', 'D', 'DD'], 5, 4) == 0.9073

Wozu das gut ist: Diese Art mehrstufiger Wahrscheinlichkeitsberechnungen ist beliebt in Programmierwettkämpfen und Wahrscheinlichkeitslehrbüchern.

Bedingungen:
1 ≤ units_one ≤ 10
1 ≤ units_two ≤ 10
2 ≤ len(dice_description) ≤ 10
Es gibt mindestens einen Angriff auf einem Würfel.
Auf einer Seite sind höchstens drei Zeichen.
